git 的使用


工作区---->编辑器中可以看到的文件
暂存区----->通过git add 将文件添加实现
远程分支---->通过git push 将文件提交到远程分支

git add . 和 git add -A; 在git2.x 中两者等同了

git add . 可以把所有的文件添加进去---包括修改和新增，但是不包括 删除 的文件
git add -A 可以把所有的变化提交上去； 修改 新建 删除

git commit

git commit -m 'xxx'---->只能将 暂存区(只能通过add把文件添加到暂存区) 的内容提交到当前分支，
如果想把最新的代码提交需要 再次执行 git add .
git commit -am 'xxx'---->可以将直接把文件绕过暂存区，直接提交到分支上；
总的来说 idea 的commit 相当于 命令行的 git commit -m '' 与 git add. 的联合作用； 这也就是说 使用命令行最好 使用git commit -am ''

新建和切换分支

常用的创建分支：git branch 分支名
git checkout 分支名

删除分支

git branch -d 分支名---->普通删除分支
git branch -D 分支名 ---->强制删除分支
先执行 git branch -d 分支名称 再执行 git push origin :分支名 ---->删除远程分支

回复分支

git log----->查看提交记录
git reflog----->查看 所有 的提交记录
git branch 分支名 hash值------>恢复某个分支

删除暂存区的内容

git rm --cached 文件名称 ----删除暂存区的内容，但是工作区的内容不变
git rm 文件名称 ----工作区和暂存区的内容全部删掉

将暂存区回复到暂存区

git checkout 文件名

git reset

git reset hash值 --hard ----->回退到某次的提交
git reset HEAD^ --hard ---->当前分支回退一个commit git reset HEAD^^ --hard ---->当前分支回退两个commit

git stash

git stash push -u----->被修改的文件和未跟踪的文件也被存储

创建远程分支

git push origin test:test 创建远程test分支

git push

git push origin master
git push
  • 当只关联一个远程，只有一个分支时，这两个命令没什么区别。推荐使用
  • 当你关联了两个多个仓库、有多个分支时，git push可能会报错，因为它不知道要上传代码到哪里去；
如果当前本地分支在 test 分支 想要推送到 远程main分支上该怎么办呢？ 我们可以直接 执行 git push origin main:main
git rebase : 提交远程分支的一种病方法；
